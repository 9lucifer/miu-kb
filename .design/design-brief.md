# 设计简报（Design Brief）

## 项目信息
- 路径 / URL：`/Users/bytedance/.codex/memory-review`，当前页面 `http://127.0.0.1:17321/?token=...`
- 技术栈：Node.js 单文件本地服务，`bin/server.mjs` 内联 HTML / CSS / 浏览器 JS
- 日期：2026-06-30

## 目标用户
面向未来安装这套个人知识库工作流的 Codex 用户，而不只是当前机器上的个人调试页面。
页面需要像一个可分发的专业控制台：能让用户管理候选记忆、长期知识库、AI 提炼队列、自检和设置。

## 品牌调性
极简 + 科技。
整体应克制、清晰、可信，带一点 AI-native / developer console 的质感，但不能牺牲长时间审核和扫读效率。

## 硬约束（不准改）
- 现有路由和接口不能动。
- 现有功能逻辑不能动：审核、写入、删除、AI 队列、自检、设置都保留。
- 页面文案必须保持中文。
- 本地端口和 token 机制不动。

## 痛点
1. 整体看起来像临时调试页，不像可分发产品。
2. 右侧状态栏像附属块，和主内容关系不清。
3. 信息密度太散，专业控制台感不够强。

## 参考站
- https://vorpus.github.io/performativeUI/#/ —— 借鉴紧凑、强状态感、monospace 细节、清晰边界和轻微科技质感。

## 设计方向
采用“专业控制台”方向：保留浅色主界面以保证审核可读性，用更清晰的布局层级、统一控件、紧凑列表和状态面板提升产品感。
不照搬 performativeUI 的大面积暗色、营销 hero 或强动效；仅吸收它的控制台质感、状态表达和精致的组件边界。

## 本次不改的范围（out of scope）
- 不重构为独立前端工程。
- 不新增后端接口或改变数据库结构。
- 不改变候选记忆、知识库、AI 队列、自检、设置的业务语义。
- 不做暗黑主题切换，除非后续单独确认。
